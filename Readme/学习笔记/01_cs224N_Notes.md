# 1. 导论和词向量
1. 通常我们可以把语义看作是能指和所指之间的配对关系，一个符号和词语对应某个概念或事物---用词向量
   - 指示语义学：one-hot编码，可是两个向量正交，没有任何关联（局部表示，向量的特定位置会对应一个词的表示，某个词都在向量的一个特定位置上）
   - 分布语义学：了解一个词的意思可以根据其上下文的相关向量来捕捉词的含义————产生语义向量，我们把这种方式称为嵌入，高维空间中的向量，词义越相近的向量的点积越大。
2. 有什么方法实现？—— **word2vec**
   - **word2vec的预测概率公式的数学推导**；采用的梯度下降算法：Mini Batch Gradient Descent,随机梯度下降算法效果更好
   - 我们需要用随机小数值来随机化向量，接着用估算的方法遍历语料库的所有位置，然后看梯度更新向量，最后学到的词向量能很好地捕捉语义关系
   - Word2Vec（Skip-gram / CBOW）在训练时会为每个词维护两个向量：(其实就比较像tranformer中的Query和Value矩阵的作用)，**原因**：词在 “中心” 和 “上下文” 中的角色是不同的（比如 “苹果” 作为中心词时，上下文可能是 “吃、手机”；作为上下文词时，可能对应 “公司、水果”）。
       - 输入向量 vw​（也叫 center/input vector）：当词作为中心词时使用
       - 输出向量 uw​（也叫 context/output vector）：当词作为上下文词时使用
   - loss functions for training:
      - Naive softmax 考虑所有可能的上下文词进行数学运算
      - More optimized variants like hierarchical softmax
      - Negative sampling 负采样
        - 在之前的softmax中我们计算概率的时候分母为所有概率的词之求和，每次计算分母都要遍历所有词，计算成本极高，训练速度极慢
        - 思路是：不再计算全词表的 Softmax，而是只采样几个「不相关的词」作为负样本，用它们来近似损失函数，把问题简化为「区分正样本和少量负样本」。
   - 之前的方法：SVD（Singular Value Decomposition），矩阵分解，但是计算复杂度太高，训练速度慢，而且效果也不如word2vec；但是也发现了**线性语义**这个特点
     - **线性语义**：词向量之间的加减运算能产生有意义的语义结果，比如：king - man + woman = queen
   - 词向量的评估方法：词向量类比、词相似性评估（人为打分两个词的相似度，衡量人工判断和机器判断的相似性）
     - NLP中一个常见的下游任务就是 命名实体识别，识别各种名词的类比（地点、人名）
   - 多义词怎么办？
     - 多义词：比如 “bank” 可以指银行，也可以指河岸，所以一个词对应多个词义，所以一个词可以用bank1、bank2...聚类到不同的语义空间
     - 目前用到的方法是只学习一个向量，是不同语义的加权平均态；好处就是其实可以不用分割成那么多毫无相关的语义，有好多不同的语义其实意思还是相近的。**一个词的含义更像是在各个意义上的概率密度分布**
3. 深度学习分类器（经典任务之一）
   - 基于窗口的 NER 分类：对句子里的每个词，用它前后若干个邻居词组成一个上下文窗口（Context Window）。把这个窗口里所有词的词向量拼接成一个长向量，输入逻辑回归分类器，判断中心词是否属于某个实体类别（比如 “地点”“人名”）。**本质是把序列标注问题转化成了窗口级别的分类问题。**
   - 对于分类器来说我们采用的是监督学习，**神经网络分类器**
   - 使用pytorch构建模型的时候，需要用到交叉熵损失：cross entropy loss  目标：最大化正确类别的概率，等价于最小化该类别的负对数概率；信息论视角：交叉熵 H(p,q) 衡量真实分布 p 和模型预测分布 q 之间的 “差异”。
